# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities privately rather than in a public issue.

- Preferred: GitHub private vulnerability reporting (the repository Security tab,
  "Report a vulnerability").
- Alternatively, contact the maintainers directly.

Include a description, reproduction steps, and the affected version. You can
expect an acknowledgement within a few business days.

## Supported versions

Security fixes land on the latest release. Keep this project current with the
foundation it was generated from (`copier update`) to receive upstream fixes.
